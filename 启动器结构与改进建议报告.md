## 5. 已提供能力 vs 典型「生产级 Spring Cloud」常见缺口

### 5.1 Starter 已侧重提供的能力

- **MVC / Web**：Web Starter、全局异常（`MvcExceptionAutoConfiguration` → `GlobalExceptionHandler`）、统一结果 `ApiResult` / `ErrorResult`、业务码与 `spring.application.module-code` 等约定。
- **校验**：AOP + `ValidAspect`（存在 `Validator` / `SmartValidator` 时生效）。
- **Jackson**：脱敏模块；环境后处理器。
- **MyBatis**：通用 Mapper、动态 SQL Provider、分层 Service、PageHelper 等。
- **数据源**：JDBC + Druid；可选读写分离自动配置。
- **日志**：专用自动配置 + `logback-spring.xml`；`TraceIdFilter`（轻量，不等同完整分布式追踪）。
- **OpenAPI**：SpringDoc + Swagger UI；Nacos 元数据（条件于 classpath 存在 Nacos Discovery）。
- **配置加解密**：Jasypt。
- **微服务入口**：`@PeachCloud`（业务侧通常需显式引入 Nacos Discovery）。
- **`peach.api.context`**：形态路径前缀与启动期类级注解校验（见第 6 节）。

### 5.2 常见缺口（本 artifact 通常不包含或仅点到为止）

- **服务间调用与负载均衡**：OpenFeign / LoadBalancer / gRPC 等（多由业务或其它 Peach 模块承担）。
- **配置中心**：仅 optional Nacos Config，无通用 Spring Cloud Config 抽象。
- **熔断限流**：无 Resilience4j / Sentinel 等 starter 级集成。
- **标准可观测性栈**：无 Micrometer Registry、OpenTelemetry 全套；以日志 + TraceId 为主。
- **Actuator**：`pom` 中通常未引入 `spring-boot-starter-actuator`。
- **消息、缓存、分布式锁**：无 Kafka/Rabbit、Redis、Redisson 等默认集成。
- **网关**：由独立 `peach-gateway` 等工程承担。

## 6. `peach.api.context` 行为与排除方式

### 6.1 命名关系（易混）

| 维度 | 值 |
|------|-----|
| 配置前缀 | `peach.api.context` |
| 配置属性类 | `org.peach.common.mvc.api.context.properties.ApiContextProperties` |
| 自动配置类 | `org.peach.common.mvc.api.context.autoconfigure.ApiContextAutoConfiguration` |

### 6.2 行为摘要

- **`peach.api.context.enable`**：默认 **`true`**（`matchIfMissing=true`）。为 `true` 时，为「表面」业务控制器注册 **`/admin`、`/app`、`/openapi`** 之一作为路径前缀。
- **`peach.api.context.path`**：类上**未**标注 `@AdminApi` / `@AppApi` / `@OpenApi` 时的默认形态；默认 **`admin`**；`enable=true` 时 **不应为空**。
- **形态注解**：**仅支持类上**；方法上若出现形态注解，启动校验会失败。
- **`ApiContextMappingValidator`**：与 `enable` 解耦注册；`enable=false` 时仍会校验「方法上不得有形态注解」等；`path` 为空的报错逻辑与 `enable` 条件相关，以源码为准。

### 6.3 关闭或排除

1. **仅关闭路径前缀**：`peach.api.context.enable: false`。前缀相关 `WebMvcConfigurer` Bean 不注册；**校验器 Bean 仍可能执行**（误在方法上标注仍会失败）。
2. **整段不要该能力**：  
   `spring.autoconfigure.exclude=org.peach.common.mvc.api.context.autoconfigure.ApiContextAutoConfiguration`

## 7. 优化建议（分层，可渐进落地）

### 7.1 安全

- 明确文档：当前 **`spring-security-core` 不等于完整 Spring Security Web / OAuth2 资源服务器**；生产鉴权与 Swagger 暴露策略建议在网关 + 业务 Security 中收口。

### 7.2 可观测性

- 若需对齐行业默认：在 BOM 或业务中引入 **Actuator + Micrometer + OTLP/Jaeger** 等；说明 `TraceIdFilter` 与网关 trace header 的对应关系。

### 7.3 配置与文档

- **README 自动配置表**与 **`AutoConfiguration.imports`** 对齐；补充 **`peach.api.context`** 与 **`ApiContextAutoConfiguration`** 说明及 **exclude 全限定类名**。
- **升级说明**：`peach.api.context.enable` 默认 `true` 会改变 URL 前缀行为，迁移文档中写明「关闭前缀」与「exclude 自动配置」两种路径。

### 7.4 测试

- 为 **`ApiContextAutoConfiguration`**、**`GlobalExceptionHandler`**、**`enable=false` 时校验器行为** 等补充最小集成测试矩阵。

### 7.5 自动配置顺序

- 对依赖 `WebMvc` / `RequestMappingHandlerMapping` 的配置，评估 **`@AutoConfigureAfter`** 或 `imports` 顺序，降低启动顺序边界问题概率。

### 7.6 性能与数据访问

- PageHelper + 大列表：注意 count 与 N+1；读写分离下 **只读事务路由** 等典型坑在文档中点名。

### 7.7 命名与职责（可选）

- **`ApiContextValidator`** 同时承担「表面判定 + 类形态解析」，名称易被理解为「仅校验」；长期可在文档中说明职责，或考虑更贴切的类名（属重构范畴）。

| Feign / LoadBalancer | 无 | 业务或 cloud BOM |
